# Architectural Decision Records

Use this folder only for important decisions that affect architecture, database design, security, deployment, or team workflow.

Do not create ADRs for minor implementation details.

## ADR Template

```markdown
# ADR-000: Title

Date:
Status: Proposed | Accepted | Superseded

## Context

What problem or decision prompted this ADR?

## Decision

What did the team decide?

## Consequences

What tradeoffs, follow-up work, or risks does this create?
```
