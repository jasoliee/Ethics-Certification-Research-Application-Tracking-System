# File Storage Strategy

ECRATS handles sensitive research documents, payment proofs, anonymized reviewer files, and certificates. These files must not be stored under `public/`.

## Storage Locations

Use private Laravel storage paths such as:

```text
storage/app/private/ecrats/applications/{application_code}/documents
storage/app/private/ecrats/applications/{application_code}/payment-proofs
storage/app/private/ecrats/applications/{application_code}/anonymized
storage/app/private/ecrats/applications/{application_code}/revisions/{cycle}
storage/app/private/ecrats/certificates/{control_number}
```

Exact paths should be generated by one storage service, not scattered through controllers.

## Upload Rules

- Validate required documents server-side.
- Validate MIME type and extension.
- Validate maximum file size.
- Store original uploaded filename separately from stored path.
- Store file size and MIME type.
- Track document versions.
- Mark which document version is current.

## Access Rules

- Applicants access only their own submitted files and released certificates.
- Advisers access only applications assigned to them during initial review.
- Reviewers access only reviewer-visible anonymized files for assigned applications.
- RES users access files needed for screening, assignment, release, certificates, and audit.
- Public web paths must not expose private files.

## Certificate Rules

- Generated certificates are private files.
- QR token access must resolve through an authorized route.
- Verification logs should record valid, not found, revoked, expired, and unauthorized attempts.
